public interface Runnable{

public void run();

}

1、每一个线程有它自己的上下文，即每个线程对象都有属于自己的内存空间存放自己的运行状态

start()与run的区别：

当实例化一个Thread对象并执行start()之后，线程进入就绪(Runnable)状态

进入了就绪状态，并不表示就开始执行run()方法，而是被排入了一个队列中，由线程调度器从队列中取出线程来轮流执行

几种让运行中的线程进入Not Runnable状态，或是blocked状态或阻塞状态：

调用sleep()

调用wait

等待I/O完成

线程调度器将不分配执行时间给这个线程，直到以下的几个情况让线程回到Runnable状态：

线程调用notify()

线程调用notifyAll()

线程呼叫interrupt()

sleep()结束

I/O完成

当线程执行完成（或发生例外）而离开run()方法，则线程执行完毕，进入Dead状态，可以使用isAlive()方法来测试线程是否存活。

Daemon 守护线程

public static void main(String[] args){

Thread thread = new Thread\{

//匿名类的写法

new Runnable()\{  


    public void run()\{  


        while(true)\{

    \}

//设定为Daemon线程

thread.start();

}

线程已经运行到最后一个语句了，应该是工作结束的时候了，

2、线程有其优先权，由1（Thread.MIN_PRIORITY）到10（Thread.MAX_PRIORITY），预设是 Thread.NORM_PRIORITY（5），您可以使用Thread的setPriority()方法来设定线程的优先权，设定必须在1到10之间，否则会丢出IllegalArgumentException。

优先权高的线程会先被执行完毕，然后才会轮到优先权低的线程，如果优先权相同，则输流执行（Round-robin方式）。

线程的休眠与唤醒：线程的休眠指线程暂时处于等待的一种状态，调用Thread类的sleep()方法

若线程因为执行sleep()或是wait()而进入Not Runnable状态，可以使用interrupt(),，而程序会丢出InterruptedException例外，因而使得线程离开run方法

终止线程的唯一方法：是使run()执行结束

结束run()运行的方法：

设置线程运行结束的标志(stopFlag)

线程终止后，不能通过start方法

while(!stopFlag){……}

while(!interrupted)

两种方法的比较

①使用Runnable接口

  • 可以从其他类集成
  • 保持程序风格的一致性
  • 更灵活、更常见

②直接继承Thread类

  • 不能再从其他类集成
  • 编写简单，直接作为线程操纵

3、线程常用操作方法

在多线程中所有的操作方法实际上都是从Thread类开始的，所有的操作基本都在Thread类之中

取得和设置线程名称

  • 在Thread类中，可以通过getName()方法取得线程的名称，通过setName()方法设置线程的名称
  • 线程的名称一般在启动线程前设置，但也允许为已经运行的线程设置名称，允许两个Thread对象有相同的名称，但为了清晰，应该尽量避免这种情况的发生
  • 若程序没有为线程指定名称，系统会自动分配名称

currentThread()方法：取得当前正在运行的线程

isAlive()方法：用于判断线程是否在运行

线程的强制运行：在线程操作中，可以使用join()方法让一个线程强制运行。在该线程运行期间，其他线程无法运行，必须等待此线程执行完成之后才可以继续执行。

线程的礼让: 使用yield()方法将一个线程的操作暂时让给其他线程执行

线程的优先级（默认是5，可以通过setPriority实现）

4、信息在多线程下共享时，容易因为多个线程同时更新同一个对象的信息，而造成信息冲突

两个以上线程共享同一个数据取进行读写操作，则该共享数据区就是一个临界区

由于多个线程执行顺序的不同，会影响临界区中变量的值或程序输出，因此对临界区的访问需要特殊处理

5、互斥锁

  • 对于临界区（由两个及以上线程共享内存的区域）变量的读写，对于同一个线程来说应该是一个原子操作
  • 在Java中引入了“对象互斥锁”来实现原子操作

实现方案：

  • 每一个对象有一个“互斥锁”
  • synchronized关键字用来获取对象的互斥锁，当某个对象用synchronized修饰时，表明该对象在任一时刻只能由一个线程访问
  • 当线程A想要访问临界区中的共享变量时，它必须要获得临界区对象的“对象锁”。若获取成功，则可访问临界区，否则进入等待区
